# LSTM 기반 블랙박스 사고 영상 자동 수집 및 비식별화 시스템

학회(KHUDA) 컨퍼런스 우수상 수상

---

## 1. 개요 (Summary)

본 시스템은 블랙박스 영상에서 사고 발생 여부를 자동 분류하고, 사고 시점 전후의 핵심 구간을 클립으로 추출하여 개인정보(얼굴 및 번호판)를 비식별화하는 서비스입니다.

### 배경 · 문제 인식

사고 입증에 블랙박스 영상이 중요하지만, **자차 블랙박스만으로는 한계**가 있습니다. 사각지대·영상 손상·영상 미존재 등으로 **단독 입증이 어려운 경우**가 있고, 목격 차량 등 **제3자 관점의 영상**이 선행 사례에서도 도움이 된 바 있습니다.

### 본 프로젝트의 방향

위와 같은 맥락에서, **사고 구간을 자동으로 찾고·수집하며 공유 가능한 형태로 가공**하는 것을 목표로 합니다.

![해결 방향](img/해결방안.png)

- **AI 기반 감지:** 임베디드·서버 파이프라인에서 사고 여부를 자동 판별합니다.
- **자동 수집·분류:** 사고로 판단된 구간을 클립 단위로 추출·분류합니다.
- **비식별화:** 번호판·얼굴 등 개인정보 영역을 자동 블러 처리합니다.
- **공유:** 처리된 영상을 웹을 통해 업로드·열람·공유할 수 있도록 구성합니다.

### 모델·시스템 요약

**시퀀스 입력 전환:** 차량 사고의 동적 특성을 반영하기 위해 입력 단위를 단일 프레임에서 시퀀스(클립) 단위로 전환하였으며, 이를 통해 사고 분류 성능(Acc/F1)을 기존 69 수준에서 97 수준으로 개선하였습니다.

**ResNet18 백본 수정:** 표준 ResNet18에서 **Layer 4를 제거**하고 `AdaptiveAvgPool2d`로 공간 차원을 1×1로 압축하여 프레임당 **256차원** 특징 벡터를 얻도록 구성했습니다. **Layer 1–2는 Freeze**로 고정하고, 블랙박스 도메인 적응을 위해 **Layer 3만 선택적 Unfreeze**로 파인튜닝할 수 있게 설계했습니다. 추출된 시퀀스 특징은 **LSTM(hidden=256)** 으로 시간축 패턴을 모델링한 뒤, **Sigmoid 기반 사고 확률**과 **임계값(0.8)** 분기로 후속 비식별화·웹 파이프라인을 트리거합니다.

## 3. 시스템 파이프라인 (System Pipeline)

1. **데이터 입력 및 전처리** — 블랙박스 영상 수집 및 프레임 추출
2. **특징 추출** — CNN Backbone을 통한 프레임별 특징 추출
3. **시퀀스 구성** — 슬라이딩 윈도우 방식을 활용한 클립 생성
4. **사고 분류** — LSTM 기반 시퀀스 분류 및 사고 여부 판별
5. **비식별화** — 사고 구간 내 얼굴 및 번호판 탐지 후 블러(Blur) 처리
6. **출력** — 결과 영상 저장 및 웹 기반 UI 서빙

## 4. 주요 엔지니어링 포인트 (Engineering Points)

### 4.1. 시계열 특성 반영을 통한 오탐 방지

- **문제:** 단일 프레임 판별 시 급정거 및 급격한 방향 전환을 사고로 오인하여 정확도가 60% 후반에 머무름.
- **해결:** LSTM 입력 단위를 30프레임 클립으로 변경하여 시간 축의 동적 패턴을 학습하도록 함.
- **결과:** 학습 데이터 구조 변경만으로 Acc/F1 스코어를 97% 수준으로 향상.

### 4.2. 엣지 디바이스 환경을 위한 모델 경량화

- **문제:** 제한된 하드웨어 리소스 내에서 실시간 추론 필요.
- **해결:** ResNet18의 Layer 4를 제거하여 파라미터 수를 50% 절감.
- **학습 전략:** 범용 특징 추출 층(Layer 1–2)은 Freeze하고, 도메인 특성 학습을 위해 Layer 3만 Unfreeze하여 파인튜닝 진행.
- **결과:** 연산 효율 확보와 동시에 Accuracy/F1 스코어 99% 달성.

### 4.3. 비동기 트리거 기반 파이프라인 설계

- **문제:** 모든 프레임에 비식별화 모델(YOLOv8, dlib)을 적용할 경우 처리 지연 발생.
- **해결:** 사고 확률이 임계값(0.8)을 초과하는 특정 윈도우에서만 비식별화 모듈이 동작하도록 Early Stop 로직 및 비동기 트리거 설계.
- **결과:** 사고 미발생 구간의 연산을 차단하여 전체 추론 속도 개선.

### 4.4. 전처리 Squash 전략 적용

- **전략:** 리사이즈 시 패딩(Padding)을 사용하지 않고 화면을 꽉 채우는 Squash 방식을 채택하여 추가 연산량 억제.
- **근거:** 형태 왜곡이 발생하더라도 시계열 모델의 사고 패턴 파악에는 지장이 없음을 확인하여 픽셀의 동적 변화량 보존에 집중.
